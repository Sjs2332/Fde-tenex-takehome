================================================================================
PROJECT OVERVIEW
================================================================================

Name: Tenex Intelligence
Type: AI-powered calendar management assistant
Stack: Next.js 16.1.6 (App Router), React 19, TypeScript 5, Vercel AI SDK v6,
       Firebase Auth + Firestore, Google Calendar API, Gmail API, Tailwind CSS 4
LLM: OpenAI GPT-4o-mini via @ai-sdk/openai

The app is a single-page dashboard with:
- Left sidebar: navigation + user menu (logout only)
- Center: AI chat interface with markdown rendering, tool status indicators,
  interactive widgets (stats, schedule, email drafts), and quick action chips
- Right sidebar: mini calendar, event timeline, "Schedule New" button, help popup
- Header: breadcrumb + event search bar with dropdown

SECURITY CONFIGURATION
================================================================================

File: next.config.ts
- Content-Security-Policy: whitelisted origins for scripts, styles, fonts,
  images, API connections (Google, Firebase, Gmail)
- X-Content-Type-Options: nosniff
- X-Frame-Options: SAMEORIGIN
- X-XSS-Protection: 1; mode=block
- Referrer-Policy: strict-origin-when-cross-origin
- Permissions-Policy: camera=(), microphone=(), geolocation=(), interest-cohort=()
- Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
- frame-ancestors: self
- frame-src: Firebase auth popup + Google accounts

Token Security:
- Google access tokens stored in HttpOnly cookies (server-side)
- Cookie flags: httpOnly=true, secure=true (prod), sameSite=lax, maxAge=3600
- Token NEVER sent to the LLM — captured in server-side closure
- localStorage fallback maintained for client-side Google API calls (sidebar)

Firestore Rules:
- All data scoped to users/{userId}
- request.auth.uid == userId required for all operations
- Default deny on all unmatched paths

================================================================================
AI AGENT BEHAVIOR
================================================================================

System Prompt Parameters:
- Receives current date/time as temporal anchor
- Has 3 tools: get_events, create_event, delete_event
- Mandatory conflict check before every create_event
- Rescheduling protocol: delete old → check new time → create new
- Widget triggers: AI auto-appends [WIDGET: STATS/SCHEDULE/EMAIL] at end of
  response. Markers are invisible — UI strips them and renders widgets instead.
  The AI never explains the markers or mentions them in the text.
- Calendar citations: [[Event Title]] → clickable buttons
- Personality: direct, no filler, uses markdown, proactive about issues

Tool Execution Flow:
1. User sends message → /api/chat (POST)
2. AI SDK streamText() with tools
3. LLM decides to call tool(s) → tool.execute() runs server-side
4. Tool makes Google API call with user's token from cookie
5. Result streamed back to client
6. Client detects tool completion → updates sidebar + logs to Firestore

Max Steps: 5 (stopWhen: stepCountIs(5))
Model: gpt-4o-mini

================================================================================
DATA FLOW
================================================================================

Authentication:
1. User clicks "Sign in with Google" on landing page
2. Firebase Auth popup → Google OAuth with calendar + gmail scopes
3. On success: access token sent to POST /api/auth/session
4. Server sets HttpOnly cookie "gat" with the token
5. Token also saved to localStorage (fallback for client-side calendar fetch)
6. User redirected to /app

Chat Message:
1. User types or clicks quick action → sendMessage()
2. DefaultChatTransport sends to /api/chat with Authorization header
3. Server reads token from cookie (primary) or header (fallback)
4. streamText() processes with tools
5. Streamed response rendered by MessageRenderer
6. On stream end: conversation saved to Firestore, activity logged

Calendar Sync:
1. CalendarProvider fetches events on mount via CalendarService
2. Both ChatInterface and RightSidebar consume shared context
3. When AI creates/deletes event → refetch() called → all consumers update
